Project Overview

Challenge:

There existed no landing page and administrative data view for the Data and Specimen Hub (DASH).

Solution:

Design a responsive landing page and administrative view in order to facilitate an emaculate user experience for researchers.

Role:

Content strategy, Research, Wireframing, Prototyping

Tool:

Illustrator

Landing Page & User Interviews

My team and I were given the task of creating a landing page for the DASH site, since the initial landing page showed default search results (imagine going to Google.com and it showed you default search results). The site was very content heavy naturally, considering it was a site made to store and access research data. I conducted user interviews with five researchers that use the site frequently to access or upload data. I needed to know what sort of layout would best be suited for the most frequently used functionality on the site.

The most important features users expressed were:

  1. Search functionality
  2. View the latest studies
  3. Short links to Explore studies, share data, and the tutorial

Administrator Page Overview

My team and I were given the task of creating an administrative site for the DASH site. The administrative site would manage the users on the site and keep track of how the user. It would also monitor study content information.

The most important features for the admin page:

  1. users/profiles information
  2. Study content information
  3. Entry/Exit points on the site
  4. Top searches
